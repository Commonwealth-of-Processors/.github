# コモンウェルスへようこそ！

**空前絶後の半導体時代**、今や学生・ホビイスト・社会人の各々が、各々の目的のために **プロセッサ** の設計・改造を行っています。

ただプロセッサと一概に言っても、実際にはLinuxが動作するような **AP(Application Processor)** や、電子制御に用いるような **MCU(microcontroller unit)** 、そして複数のコア・IPを備えた **SoC(System on Chip)** と、その種類は数多に渡ります。

しかし、これらの分野間、あるいはプロジェクト間で **設計資源（回路・IP・知見）が共有されることは稀であり**、貴重なリソースが腐っているのが現状です。

**Commonwealth of Processors** では、プロセッサ設計における **共通財(Common-Goods)** を中心に、AP・MCU両面の設計・実装を行います。それらの開発過程や応用で生まれる新たな資源を、AP・MCU双方へスムーズに活用することで、 **デザインの発展** と **異分野間での資源の共有** を目的としています。
