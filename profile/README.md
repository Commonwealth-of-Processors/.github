# コモンウェルスへようこそ！


**空前絶後の半導体時代**、今や学生・ホビイスト・社会人の各々が、各々の目的のために **プロセッサ** の設計・改造を行っています。

ただプロセッサと一概に言っても、実際にはLinuxが動作するような **AP(Application Processor)** や、電子制御に用いるような **MCU(microcontroller unit)** 、そして複数のコア・IPを備えた **SoC(System on Chip)** と種類は数多に渡り、そして種類の数だけ **設計資源** が存在します。

しかし、ある分野で生み出された資源が、別の分野で活用される事は極めて稀となっているのが現状です。

つまりAPの設計で生み出された回路・IP・知見がMCUで生かされる事は稀であり、また別のAPで利用しようとすれば、慣れない外国語を用いてドキュメントを読み、コミュニティで議論をする新たな困難に直面する事になります。

**Commonwealth of Processors** では、プロセッサ設計における **共通財(Common-Goods)** を中心に、AP・MCU両面の設計・実装を行い、それらの開発過程や応用で生まれる新たな資源のAP・MCU双方へのスムーズな活用による **デザインの発展** と **異分野間での資源の共有** を目的としています。
